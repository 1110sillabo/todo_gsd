<task type="auto">
  <name>Task 1: Create TaskNode scene and script</name>
  <files>src/ui/task_node.tscn, src/ui/task_node.gd</files>
  <action>
    Create a GraphNode-based scene `TaskNode.tscn`. The script `task_node.gd` should:
    - Hold a reference to a `TaskResource`.
    - Update its title and content based on the resource.
    - Connect to the `position_offset_changed` signal to update `task_resource.graph_position`.
    - Emit a custom signal `task_moved(task: TaskResource)` when the position changes.
  </action>
  <verify>
    Run a minimal script that instances TaskNode, sets a TaskResource, and checks if position changes reflect in the resource.
  </verify>
  <done>TaskNode displays resource data and syncs position.</done>
</task>

<task type="auto">
  <name>Task 2: Create TaskBox scene and script</name>
  <files>src/ui/task_box.tscn, src/ui/task_box.gd</files>
  <action>
    Create a `TaskBox.tscn` scene with a `GraphEdit` node and an "Add Task" `Button`.
    The script `task_box.gd` should:
    - Use `PersistenceManager` to load all existing tasks on `_ready()`.
    - For each task, instance a `TaskNode` and add it as a child of `GraphEdit`.
    - Handle the "Add Task" button: create a new `TaskResource`, save it via `PersistenceManager`, and add a `TaskNode`.
    - Connect `task_moved` from nodes to trigger `PersistenceManager.save_task()`.
    - Enable `show_zoom_label` and `show_grid` for mobile-friendly navigation.
  </action>
  <verify>
    Manual verification or a SceneRunner test that proves GraphEdit contains the expected number of nodes after loading.
  </verify>
  <done>TaskBox loads existing tasks into the graph and supports adding new ones.</done>
</task>

</tasks>
